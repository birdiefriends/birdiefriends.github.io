🌐 Dev-67 update — new external dependency: the Worker now makes outbound server-side calls to Open-Meteo (open-meteo.com), a free, no-key weather service — the first third-party API besides OneSignal it calls directly rather than just proxying. Two endpoints: the Archive API (archive-api.open-meteo.com) returns historical daily high/low temp, wind, precipitation, and conditions for a venue's lat/lng + date, fired by POST /weather/capture right after a player's first photo or scorecard for an event; and the Geocoding API (geocoding-api.open-meteo.com) is a best-effort fallback that resolves a venue name to coordinates when Brian hasn't set them manually in Venue Manager, caching the result in geocode_cache so it's a one-time lookup per venue name. Both calls are fire-and-forget from the portal's side and fail silently/self-heal on the next capture attempt — no retry job, no user-facing error path. See the D1 Schema entries above (Entry 20) for the tables involved.
